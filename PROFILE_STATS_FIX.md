# Исправление статистики профиля

## Проблема
На странице профиля отображались ложные (хардкоженные) данные:
- 12 заказов
- ₽24K потрачено
- 8 избранное

## Решение

### 1. Создан новый хук `useProfileStats`
Файл: `src/hooks/useProfileStats.ts`

Хук получает реальные данные из базы:
- **Количество заказов** - из таблицы `orders` по email пользователя
- **Общая сумма** - сумма всех заказов пользователя
- **Количество лайков** - из таблицы `likes` по telegram_id

### 2. Обновлена страница профиля
Файл: `src/pages/ProfilePage.tsx`

Изменения:
- Добавлен импорт `useProfileStats` и `formatPrice`
- Заменены хардкоженные значения на реальные данные из хука
- Добавлен индикатор загрузки (`...`) пока данные загружаются
- Изменения применены как для мобильной, так и для десктопной версии

### 3. Формат отображения
- **Заказов**: число (например, 0, 5, 12)
- **Потрачено**: форматированная сумма (например, ₽0, ₽2 450, ₽24 000)
- **Избранное**: число (например, 0, 3, 8)

## Результат
Теперь на странице профиля отображается только актуальная информация из базы данных. Если у пользователя нет заказов или лайков, будут показаны нули.

## Дополнительно
Страница "Мои заказы" (`MyOrdersPage.tsx`) уже использовала реальные данные, изменений не требовалось.
